# Лабораторная работа №5

## Описание
Реализован семантический поиск по датасету StackSample с использованием SentenceTransformers и Chroma DB. Эмбеддинги генерируются для объединённого текста вопроса и ответа, хранятся в Chroma, выполняется поиск по смысловому сходству.

## Метрики
- Precision@5 (Semantic Search): 1.00
**Примечание**: Использованы запросы, связанные с Python (матрицы и unit testing).

## Плюсы и минусы
- **Плюсы**: Учёт семантики, поддержка синонимов, высокая релевантность для сложных запросов.
- **Минусы**: Ограниченная выборка (371 вопрос), высокие требования к памяти, медленная генерация эмбеддингов.

## Сравнение с ЛР4
Семантический поиск лучше учитывает смысл запроса, находя релевантные вопросы даже без точного совпадения слов. Например, для 'how to sort a list in python' находятся вопросы по Python, но не точные из-за ограничений выборки.
Размер хранилища Chroma: 1.93 MB. Chroma использует ANN-индексы для быстрого поиска.

## Проблемы и решения
- Обнаружены некорректные Id (например, ")") в Questions.csv, устранены фильтрацией по регулярному выражению `r'^\d+$'`.
- Удалены дублирующиеся секции кода для предотвращения ошибок вроде `DuplicateIDError`.
- Ограниченная выборка (364 вопроса) привела к низкой релевантности для запроса "how to sort a list in python".

## Анализ результатов
- Precision@5: 1.00 для запросов "python matrix operations" и "python unit testing", что подтверждает высокую точность семантического поиска.
- Для "how to sort a list in python" релевантные вопросы отсутствуют в выборке, что ограничивает результаты.
- Chroma DB занимает 1.93 MB, используя HNSW-индексы для ANN-поиска, что обеспечивает высокую скорость.

## Сравнение с ЛР4
Семантический поиск превосходит TF-IDF/BM25 в учёте синонимов и контекста (например, "matrix" и "numpy").
Однако ограниченная выборка снижает релевантность для некоторых запросов по сравнению с ЛР4, где полнотекстовый поиск мог находить частичные совпадения.
